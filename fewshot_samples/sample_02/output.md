## 📌 n8n 是什么？为什么选它？

**n8n**（读作 "n-eight-n"）是一款**开源免费、可自托管**的自动化工作流引擎。简单来说，就是让你用**拖拽节点**的方式，把不同的应用和服务串起来，实现任务自动化，而不需要写复杂的代码。

### 核心优势一览

| 特性 | n8n | 商业工具（如 Zapier） |
|------|-----|----------------------|
| **开源免费** | ✅ MIT 协议 | ❌ 收费 |
| **数据自主** | ✅ 部署在你自己的服务器 | ☁️ 数据存第三方 |
| **自定义能力** | ✅ 支持 JavaScript | ⚠️ 功能受限 |
| **集成数量** | ✅ 300+ 原生插件 | ✅ 也很多但有限制 |

**一句话总结**：想要**数据完全掌控**、**零成本**、还能**自由扩展**？选 n8n 就对了！

### 典型应用场景

- 🔄 **定时同步数据**：每天自动把表格数据同步到数据库
- 📧 **事件触发通知**：表单提交后自动发邮件/钉钉告警
- 🤖 **AI 智能处理**：接入 DeepSeek/GPT 实现智能对话
- 🔗 **跨平台打通**：连接飞书、企业微信、Notion 等

## 🛠️ 第一部分：环境准备（Docker 安装）

在部署 n8n 之前，我们需要先安装 Docker。如果你已经装好了，可以跳过这步。

### 步骤 1：卸载旧版（如果有）

```bash
yum remove docker docker-client docker-client-latest docker-common docker-latest-logrotate docker-logrotate docker-engine
```

### 步骤 2：安装依赖包

```bash
yum install -y yum-utils device-mapper-persistent-data lvm2
```

### 步骤 3：添加国内镜像源（重要！）

```bash
yum-config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
```

> 💡 **为什么要加国内源？** 因为直接访问 Docker Hub 在国内会非常慢甚至失败。这一步能帮你节省大量时间！

### 步骤 4：安装并启动 Docker

```bash
# 安装 Docker
yum install -y docker-ce docker-ce-cli containerd.io

# 启动并设置开机自启
systemctl start docker
systemctl enable docker

# 验证安装成功
docker --version
```

看到版本号就说明装好了！

### 步骤 5：配置镜像加速器（必做！🔴）

这是**最关键的一步**！不配置的话拉取 n8n 镜像会失败或极慢。

创建或编辑 `/etc/docker/daemon.json`：

```json
{
    "registry-mirrors": [
        "https://docker.xuanyuan.me",
        "https://docker.m.daocloud.io",
        "https://docker.imgdb.de"
    ]
}
```

然后重启 Docker 服务：

```bash
systemctl daemon-reload
systemctl restart docker
```

✅ 到这里，Docker 环境就准备好了！

## 🚢 第二部分：一键部署 n8n

### 步骤 1：拉取 n8n 最新镜像

```bash
docker pull n8nio/n8n:latest
```

### 步骤 2：创建数据持久化目录

这样即使容器删除了，你的工作流数据也不会丢失：

```bash
mkdir -p ~/.n8n
chmod -R 777 ~/.n8n
```

### 步骤 3：启动 n8n 容器

```bash
docker run -d \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  -e N8N_BASIC_AUTH_ACTIVE=true \
  -e N8N_BASIC_AUTH_USER=admin \
  -e N8N_BASIC_AUTH_PASSWORD=YourStrongPassword123! \
  -e TZ=Asia/Shanghai \
  -e N8N_COOKIE_SECURE=false \
  -e N8N_SECURE_COOKIE=false \
  --restart always \
  n8nio/n8n:latest
```

#### 🔍 关键参数说明

| 参数 | 含义 | 为什么重要 |
|------|------|-----------|
| `-p 5678:5678` | 映射端口到主机 | 浏览器通过这个端口访问 |
| `-v ~/.n8n:/home/node/.n8n` | 数据卷挂载 | **保证数据持久化** |
| `N8N_BASIC_AUTH_*` | 基础认证配置 | **必须修改默认密码！** |
| `TZ=Asia/Shanghai` | 时区设置 | 让日志时间正确显示 |
| `COOKIE_SECURE=false` | 关闭安全 Cookie | **避免登录问题** |
| `--restart always` | 自动重启策略 | 服务器重启后自动恢复 |

> ⚠️ **安全提醒**：请务必将 `N8N_BASIC_AUTH_PASSWORD` 改成你自己的强密码！示例中的密码只是演示用途。

### 步骤 4：验证运行状态

```bash
docker ps | grep n8n
```

看到容器状态是 `Up` 就成功了！

打开浏览器访问：**http://你的服务器IP:5678**

首次登录需要注册账号，按照提示填写邮箱和密码即可。

## 🤖 第三部分：实战一 —— 接入 AI 对话（DeepSeek）

接下来我们要让 n8n 变得更智能——接入 **DeepSeek AI**，实现对话功能！

### 准备工作：获取 API 密钥

1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/) 注册账号
2. 登录后进入「API 密钥」页面
3. 点击「创建新的 API 密钥」，命名为 `n8n`
4. **⚠️ 重要：密钥只显示一次，立即复制保存好！**

> 💡 **好消息**：DeepSeek 提供免费的 API 调用额度，学习和测试完全够用！

### 搭建第一个工作流

#### 1. 创建新工作流

在 n8n 主界面点击 **「Create Workflow」**

#### 2. 添加触发器节点

点击画布上的 **「+」号**，搜索并选择 **「Manual Trigger」（手动触发器）

这就是我们工作流的入口——手动点击就会执行。

#### 3. 添加 AI Agent 节点

点击触发器右侧的 **「+」号**，搜索 **「AI Agent」**

在配置界面：
- **Credentials（凭证）**：选择 **「Create New Credential」**
- **类型选择**：**DeepSeek**
- **填入 API Key**：粘贴刚才复制的密钥
- **模型选择**：`deepseek-chat`（推荐）
- **System Message（系统提示词）**：可以留空或写点角色设定

#### 4. 连接节点并测试

确保两个节点之间有连线（从触发器连向 AI Agent），然后：

1. 点击右上角 **「Test Workflow」（测试工作流）**
2. 在弹出的对话框中输入消息，比如：**「你好，请介绍一下你自己」**
3. 点击发送，等待 AI 回复

🎉 **恭喜！** 如果你看到了 DeepSeek 的回复，说明你的第一个 AI 工作流已经成功运行了！

## 🌐 第五部分：cpolar 内网穿透 —— 让外部也能访问

现在 n8n 只能在**本地局域网**访问。如果你想从外面触发 Webhook（比如手机上、远程服务器），就需要做**内网穿透**。

这里我们用 **cpolar**——一款简单高效的内网穿透工具。

### 什么是 cpolar？

cpolar 能把你的**本地服务映射到公网 URL**，无需公网 IP，无需复杂路由器配置，一条命令搞定。

**核心优势**：
- ✅ **5秒创建 HTTPS 隧道**
- ✅ **免费提供有效证书**（Webhook 友好）
- ✅ **支持断线重连**，适合长期运行
- ✅ **数据加密传输**，安全可靠

### 安装 cpolar

```bash
sudo curl https://get.cpolar.sh | sh
```

安装完成后启动服务：

```bash
sudo systemctl status cpolar
```

看到状态是 `active (running)` 就说明成功了。

浏览器访问 **http://你的IP:9200**，使用 cpolar 账号登录即可进入管理界面。

## 🔗 第六部分：配置公网地址

### 创建隧道

在 cpolar Web UI 中：

1. 左侧菜单点击 **「隧道管理」→「创建隧道」**
2. 填写参数：
   - **隧道名称**: `n8n`（自定义）
   - **协议**: `http`
   - **本地地址**: `5678`（n8n 的端口）
   - **域名类型**: 选择 **随机域名** 或 **二级子域名**
   - **地区**: **China Top**（国内节点速度更快）
3. 点击 **「创建」**

创建成功后，在左侧 **「在线隧道列表」** 可以看到生成的**公网地址**。

现在用这个地址在任何设备上访问，都能打开你的 n8n 了！🎉

### 升级为固定域名（推荐）

随机域名每次重启可能会变。如果你需要一个**永久不变的地址**：

1. 进入 cpolar 控制台 → **「预留」→「保留二级子域名」**
2. 地区选择 **China Top**
3. 设置一个子域名名称（比如 `myn8n`）
4. 返回隧道列表，点击 **「编辑」**
5. 将域名类型改为 **二级子域名**，填入刚才保留的名称
6. 点击 **「更新」**

完成！现在你有了一个**固定不变**的公网地址，随时随地都能访问本地的 n8n 了。
