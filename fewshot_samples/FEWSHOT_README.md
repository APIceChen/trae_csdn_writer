# Few-Shot 小样本提示系统

## 📌 系统概述

**Few-Shot Learning（少样本学习）** 是一种通过提供少量高质量示例来提升 AI 输出一致性和准确性的方法。

在本项目中，我们维护 **2-3 个精心挑选的高质量样本对**，AI 在执行新任务时会参考这些样本，从而：

- ✅ **保持风格一致性**：所有输出都遵循统一的 CSDN 写作风格
- ✅ **提升输出质量**：基于成功案例进行模仿学习
- ✅ **减少随机性**：避免每次输出的质量波动过大
- ✅ **加速收敛**：减少反复修改的次数

## 🎯 使用场景

### **何时使用 Few-Shot？**

1. **首次改写新类型文章时**
   - AI 自动读取最相关的样本
   - 参考其结构、语气、排版方式

2. **用户要求特定风格时**
   - 例如："写得像样本1那样简洁"
   - AI 会重点参考该样本的特征

3. **质量不稳定时**
   - 如果连续几次输出未达标
   - 回归到样本基准，重新校准

4. **新增样本时**
   - 当产出特别优秀的文章时
   - 可考虑将其纳入样本库

## 📊 样本选择策略

### **智能匹配算法**

当收到新的改写任务时，AI 应按以下优先级选择参考样本：

#### **优先级 1：文章类型匹配**

| 新文章类型 | 推荐参考样本 | 匹配度 |
|-----------|-------------|--------|
| 工具/项目推荐 | sample_01 (pyd_packer) | ⭐⭐⭐⭐⭐ |
| 技术教程/部署指南 | sample_02 (n8n) | ⭐⭐⭐⭐⭐ |
| 问题解决方案 | sample_01 或 sample_02 | ⭐⭐⭐⭐ |
| 源码分析 | sample_01 | ⭐⭐⭐ |
| 最佳实践总结 | sample_02 | ⭐⭐⭐⭐ |

#### **优先级 2：用户明确指定**

如果用户说：
- "像上次那个 pyd_packer 的风格" → 强制使用 sample_01
- "写详细一点，像 n8n 那篇" → 强制使用 sample_02

#### **优先级 3：综合多个样本**

对于复杂文章，可以融合多个样本的优点：
- 结构参考 sample_02（教程类更完整）
- 语气参考 sample_01（更活泼）

## 🔧 如何使用样本

### **AI 执行流程中的集成点**

```
用户输入 → [读取 Few-Shot 样本] → 分析原文 → 参考样本改写 → 验证 → 输出
                ↓
        选择最匹配的 1-2 个样本
        提取关键特征（结构、语气、排版）
        形成隐式的"风格模板"
```

### **具体操作步骤**

#### **步骤 1：加载样本数据**

```python
# 伪代码示例
def load_fewshot_samples(task_type):
    samples = []
    
    if task_type == "tool_recommendation":
        samples.append(load_sample("sample_01"))
    elif task_type == "tutorial":
        samples.append(load_sample("sample_02"))
    
    return samples
```

#### **步骤 2：提取特征模式**

从每个样本中提取：

| 特征维度 | 提取内容 | 权重 |
|---------|---------|------|
| **标题模式** | emoji使用、长度、句式 | 20% |
| **引入方式** | 场景描述、痛点提出、互动提问 | 15% |
| **章节结构** | 二级标题数量、逻辑顺序 | 25% |
| **代码呈现** | 块数、注释密度、语言标注 | 15% |
| **语气风格** | 口语化程度、表情符号频率 | 15% |
| **结尾方式** | 互动引导、总结要点 | 10% |

#### **步骤 3：应用特征到新任务**

在生成过程中，实时对照样本特征进行调整。

## 📝 样本质量标准

### **入选条件**

一个样本要被纳入 Few-Shot 库，必须满足：

#### **硬性指标（必须全部满足）**
- ✅ 质量评分 >= **85 分**
- ✅ 所有"必须"项检测通过率 = **100%**
- ✅ 用户满意度反馈为**正面**
- ✅ 技术内容**100% 准确**

#### **软性指标（至少满足 3 项）**
- ✅ 行文流畅自然
- ✅ 结构清晰合理
- ✅ 可读性强
- ✅ 有独特亮点
- ✅ 获得较多互动（点赞/收藏/评论）

### **样本维护规则**

1. **数量限制**：最多保留 **5 个**样本（避免过多导致混淆）
2. **定期评估**：每 **10 次任务后**回顾样本有效性
3. **优胜劣汰**：如果新样本明显优于旧样本，考虑替换
4. **多样性**：尽量覆盖不同类型的文章（工具推荐、教程、问题解决等）
5. **版本控制**：样本变更需要记录在 git 历史中

## 🚀 快速上手指南

### **对于 AI Agent**

当你被激活并执行改写任务时：

1. **自动识别任务类型**
   ```python
   task_type = analyze_task(user_input)
   # 返回值示例: "tool_recommendation" 或 "tutorial"
   ```

2. **加载对应样本**
   ```python
   samples = load_fewshot_samples(task_type)
   ```

3. **提取并应用特征**
   ```python
   style_profile = extract_style_features(samples)
   # 在生成过程中持续参考 style_profile
   ```

4. **生成后对比验证**
   ```python
   similarity = compare_with_samples(output, samples)
   # 确保 output 与 samples 的风格一致性 >= 80%
   ```

### **对于人类用户**

如果你想利用 Few-Shot 系统：

#### **方式 1：自动匹配（默认）**
直接提供 URL 或文本，AI 会自动选择最佳样本参考。

#### **方式 2：手动指定**
```
请参考样本1的风格改写这篇文章：[URL]
```

#### **方式 3：混合模式**
```
用样本2的详细程度，但保持样本1的简洁语气
```

## ⚠️ 注意事项与最佳实践

### **Do's（推荐做法）✅**

1. ✅ **始终参考样本**：即使任务简单，也快速浏览一下样本
2. ✅ **灵活调整**：不要死板照搬，要根据实际情况微调
3. ✅ **记录偏差**：如果某次输出偏离样本太多，记录原因
4. ✅ **定期更新**：根据用户反馈优化样本库

### **Don'ts（避免做法）❌**

1. ❌ **过度依赖**：样本是参考不是枷锁，要有创新空间
2. ❌ **忽略用户需求**：如果用户明确要求不同风格，以用户为准
3. ❌ **样本过多**：2-3 个高质量样本优于 10 个平庸样本
4. ❌ **一成不变**：随着时间推移，样本也需要进化

## 📈 效果评估

### **如何衡量 Few-Shot 的效果？**

#### **量化指标**

| 指标 | 衡量方式 | 目标值 |
|------|---------|--------|
| **质量稳定性** | 连续 10 次任务的评分标准差 | <= 5 分 |
| **一次通过率** | 首次即达标的比例 | >= 80% |
| **风格一致性** | 与样本的风格相似度 | >= 85% |
| **用户满意度** | 正面反馈占比 | >= 90% |

#### **质性评估**

- 输出是否"看起来像同一个人写的"？
- 是否保持了 CSDN 平台的调性？
- 是否减少了明显的风格跳跃？
