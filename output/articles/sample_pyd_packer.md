# 🔥 GitHub 开源神器推荐：pyd_packer —— 让你的 Python 项目一键"加密"打包！

大家好！👋 在日常的 Python 开发工作中，我们经常会遇到一个头疼的问题：**如何保护自己的源码？** 尤其是在项目交付或者发布给客户使用的时候，直接把 .py 源文件交出去总觉得心里不踏实... 😅

今天，我就来给大家安利一个超级实用的开源工具 —— **pyd_packer**！这个工具可以帮你把 Python 源码一键编译成 .pyd 扩展模块，既保护了代码安全，又方便部署发布！✨

> **项目地址**：https://github.com/APIceChen/pyd_packer.git

---

## 📖 什么是 pyd_packer？

简单来说，**pyd_packer** 是一个专门面向 **Windows 平台**的 Python 源码打包工具。它的核心功能就是：

- 📦 将单个 .py 文件编译为 .pyd 格式
- 📁 支持整个项目目录或 Python 包的**批量编译**
- 🔒 实现源码保护、项目交付和部署发布

最棒的是，它同时提供了 **图形界面（GUI）** 和 **命令行（CLI）** 两种使用方式，无论你是喜欢点点点的可视化用户，还是钟爱命令行的极客玩家，都能找到适合自己的操作方式！💪

---

## 🌟 七大核心亮点，每一个都超实用！

### 1️⃣ **灵活的打包模式：单文件 OR 整目录？你说了算！**

```bash
# 打包单个 Python 文件
pyd_packer build your_script.py

# 打包整个项目目录
pyd_packer build ./your_project/
```

不管你是想保护一个核心算法脚本，还是想把整个项目都打包起来，它都能轻松搞定！

---

### 2️⃣ **PySide6 图形界面：小白也能轻松上手**

对于不太熟悉命令行的朋友来说，这个工具贴心地提供了基于 **PySide6** 开发的图形化界面：

- ✅ 可视化操作，降低使用门槛
- ✅ 实时日志查看，构建过程一目了然
- ✅ 配置选项管理，方便保存常用设置
- ✅ 构建结果预览，快速检查输出产物

> 💡 **小贴士**：GUI 界面特别适合第一次使用或者需要频繁调整参数的场景哦~

---

### 3️⃣ **完整的命令行工具链：自动化脚本的好帮手**

如果你是命令行爱好者，或者需要把这个工具集成到 CI/CD 流程中，那么它的 CLI 能力绝对不会让你失望：

```bash
# 常用命令一览
pyd_packer build      # 执行打包构建
pyd_packer preview    # 预览将要处理的文件列表
pyd_packer check      # 检查环境和依赖
pyd_packer msvc-check # 专门检查 MSVC 编译器状态
```

这些命令可以轻松写入自动化脚本，实现**一键式批量处理**！

---

### 4️⃣ **智能环境检测：告别配置烦恼**

构建 .pyd 文件需要 Cython、setuptools、MSVC 编译器等依赖，手动配置起来相当麻烦... 但是！**pyd_packer 帮你搞定了这一切**：

- 🔍 自动发现 uv 管理的 Python 解释器
- ✅ 智能检测 Cython、setuptools 是否已安装
- 🔧 支持 MSVC 编译器状态检查
- 🛠️ 可自动准备和重建构建环境

再也不用为环境配置头秃了！🎉

---

### 5️⃣ **资源文件友好处理：不只是代码，还有数据！**

很多项目除了 .py 文件外，还有配置文件、图片、模板等资源文件。pyd_packer 对此也考虑得很周全：

- 📂 自动复制非 .py 资源文件
- 🗂️ 保留原有目录结构
- 🚫 支持合并默认排除规则和项目的 .gitignore
- ✨ 确保打包后的项目能正常运行

这一点在处理复杂项目时特别实用！

---

### 6️⃣ **FastAPI 项目专属支持：Web 开发者的福音**

如果你在使用 **FastAPI + Uvicorn** 开发 Web 应用，那么这个功能绝对会让你眼前一亮：

```yaml
# 自动生成的 Uvicorn 运行配置示例
host: "0.0.0.0"
port: 8000
reload: false
uvicorn_params: "--workers 4"
```

它可以自动生成 FastAPI/Uvicorn 的启动配置，支持自定义 host、port、reload 参数等，让 Web 项目的部署变得超级简单！🚀

---

### 7️⃣ **清晰的输出产物：交付无忧**

打包完成后，你会得到一套完整的输出文件：

| 产物类型 | 说明 |
|---------|------|
| .pyd 文件 | 编译后的扩展模块 |
| 资源文件 | 复制的非 Python 文件 |
| 启动脚本 | 方便快速运行 |
| 运行环境配置 | 如 Uvicorn 配置等 |
| 构建日志 | 便于排查问题 |

这套完整的产物结构让**项目交付**变得非常规范和专业！💼

## 📝 快速上手指南

### 方式一：使用 GUI 界面（推荐新手）

1. 克隆项目仓库
2. 安装依赖：`pip install -r requirements.txt`
3. 运行 GUI 程序
4. 选择要打包的文件或目录
5. 点击"构建"按钮，等待完成

### 方式二：使用命令行（推荐进阶用户）

```bash
# 第一步：克隆项目
git clone https://github.com/APIceChen/pyd_packer.git

# 第二步：进入目录并安装依赖
cd pyd_packer
pip install -e .

# 第三步：执行打包
pyd_packer build your_project/

# 第四步：查看结果
ls dist/
```
